1. vue 项目最后阶段如何打包？ #

  • 思考是否要更换为线上的接口地址，其实早期可以考虑使用环境变量来定义接口地址
  • vue.config.js 中 配置一些基础内容
const isPord = process.env.NODE_ENV === 'production'
module.exports = {
  publicPath: isPord ? './' : '/', //处理 相对路径问题
  productionSourceMap: false // 优化不要 map 文件
}
  • 执行 npm run build ，交出 dist 目录，跑路
  • 要不要考虑优化，那是看心情和需求！（一定要优化的话，做好代码备份！）

2. 如何解决 token 过期问题？ #

  • 如果响应码是 401 ，则请求获取新的 token

  • 校验是否有 refresh_token

  • 如果有 refresh_token，则请求获取新的 token

  • 如果获取成功，则把新的 token 更新到容器中

  • 如果获取失败，直接跳转 登录页面

  • 处理流程：

  1. 在 axios 的拦截器中加入 token 刷新逻辑
  2. 当用户 token 过期时，去向服务器请求新的 token
  3. 把旧的 token 替换为新的 token
  4. 然后继续用户当前的请求

3. vue 中 slot 的使用方式，以及 slot 作用域插槽的用法？ #

当组件当做标签进行使用的时候，用 slot 可以用来接受组件标签包裹的内容；插槽中可以设置默认内容， 父组件未填坑时，则显示默认插槽里面的东西；

  • 无名插槽
  1. 子组件中设置 <slot> </slot>
  2. 父组件中在子组件标签中填入需要的内容

  • 具名插槽

    1. <slot name="坑名">
    2. 父中子组件标签里 <template v-slot:坑名 简写：#坑名>

  • 作用域插槽

    1. 子组件抛出数据 ：:自定义属性=”数据” <slot :row="数据"></slot>
    2. 父组件中接收数据 ： v-slot=“自定义变量 scope “
    3. 使用： scope.row 就是接收到的数据

4. vue-router 中钩子函数都有哪些？ #

vue-router 中的钩子函数主要分为 3 类

  • 全局钩子函数
    • beforeEach
  • 单独路由独享组件
    • beforeEnter
  • 组件内钩子
  • beforeRouterEnter
  • beforeRouterUpdate
  • beforeRouterLeave

5. 路由配置项常用的属性及其作用？ #

  • path : 跳转路径
  • component : 路径相对于的组件
  • name:命名路由
  • children:子路由的配置参数(路由嵌套)
  • props:路由解耦
  • redirect : 重定向路由
