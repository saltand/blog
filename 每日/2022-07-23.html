1. webpack 打包流程 #

  1. 读取入口文件
  2. 分析入口文件，得到其所依赖的文件信息
  3. 递归读取每个文件的依赖信息，生成 AST 树，得到关系依赖图
  4. 代码转化，生成 ES5 代码

2. 原生 ajax 怎么发送 get 和 post 请求？ #

  1. 创建 XMLHttpRequest 对象
  2. ajax.open(‘请求方式’，url，true)方法创建网络请求对象，第三个参数是 true 时是异步请求，false 是同步请求，post 请求需要在发送请求之前设置 post 的请求头。
  3. ajax.send()发送请求
  4. ajax.onreadStateChange==function(){}监听请求状态

3. mvvm 和 mvc 区别是什么？ #

  • MVC 是 Model View Controller 的缩写
    • 模型层-视图层-控制器
  • MVVM 是 Model-View-ViewModel 的简写,
    • 模型-视图-视图模型
  • 在 MVVM 架构下,视图和模型是不能直接通信的,它们通过 viewmodel 来同步或者反应数据变化,实现了数据的互相绑定
    • 场景:适合数据驱动,数据操作比较多的场景
  • 在 MVC 中,Model 和 View 就是相互独立的,View 只负责页面的展示,Model 只是数据存储,就是将模型和视图之间实现代码分离,松散耦合的目的
    • 场景: 主要用于中大型项目的分层开发

4. 什么避免 v-if 和 v-for 用在一起使用？ #

Vue 处理指令时，v-for 比 v-if 具有更高的优先级, 每次渲染都会做 v-if 判断, 会造成性能浪费, 如果你有 5 个元素被 v-for 循环, v-if 也会分别执行 5 次.

5. Vuex 的 Mutation 和 Action 之间的区别是什么？ #

  1. 流程顺序
  • “相应视图—>修改 State”拆分成两部分:
    • 视图触发 Action，Action 再触发 Mutation。
  1. 角色定位
  • 基于流程顺序，二者扮演不同的角色。
    • Mutation：专注于修改 State，理论上是修改 State 的唯一途径。
    • Action：业务代码、异步请求。
  1. 限制
  • 角色不同，二者有不同的限制。
    • Mutation：必须同步执行。
    • Action：可以异步，但不能直接操作 State。
